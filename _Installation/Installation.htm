Installation of GLScene
into Embarcadero RAD Studio

The Instruction for installing GLScene library into Embarcadero RAD Studio to work with Delphi & C++Builder includes next steps:

1. Download source codes of GLScene from the URL of repository https://svn.code.sf.net/p/glscene/code/trunk to a SVN directory on your disk, e.g. D:\Library\SVN_GLScene. Use TortoiseSVN client or embedded subversion control system for RAD Studio environment in menu File | Open From Version Control… to check out the code.

You may also to get the whole current Snapshot of the trunk from code page http://sourceforge.net/p/glscene/code/HEAD/tree/  or download archive zip files with previous releases of GLScene project from the page http://sourceforge.net/projects/glscene/files/

 

2. Make a copy of the trunk in a separate directory, e.g. in the working directory D:\Library\GLSceneVCL, to prevent original sources from occasional changes. You may skip this step if you don’t need to update the code from SVN repository further.

 

3. Copy auxiliary dynamic libraries from ..GLSceneVCL\Source\external into C:\Windows\System32 directory to support 3D sounds, shaders and physics (BASS, FMOD, OpenAL, SDL, CG, ODE, Newton). Otherwise you will need to have ones in every application directory with exe file that calls an appropriate dll. To do this, please, run setup.bat in ..\GLSceneVCL\Source\external directory. Note: on 64-bit OS Windows you may need to copy *.dll  files to C:\Windows\SysWOW64 directory manually.

 

4. Setup Delphi Library Paths in Options dialog. Add the directory D:\Library\GLSceneVCL\Source to search files for GLScene’s applications as shown in the next screenshot:

 

 

 

5. Setup C++ Options for include files. Open Options dialog for Paths and Directories and add “D:\Library\GLSceneVCL\Source\include” path into “Ordered list of Include paths” to get header HPP files be found by GLScene’s CPP applications:

 

6. Setup C++ Options for libraries files. Add “D:\Library\GLSceneVCL\Source\lib” folder into “Ordered list of Library paths” to get lib/bpi files be found by GLScene’s CPP applications

 

 

 

7. Open the GLSceneVCL.groupproj file in your directory D:\Library\GLSceneVCL\Packages\ using menu item File|Open Project…(Ctrl+F11). In Project Manager window you will find the next list of projects with *.bpl extensions:

 

 

8. Check the options of Delphi Compiler to get output C/C++ files into include (HPP files) and library (BPI/LIB packages) directories for every project in GLScene’s project group (Projects|Options… Shift+Ctrl+F11). By default *dpk options are saved in *dproj files but you can type it manually and choose “Generate all C++Builder files (including package libs)”. You can also apply a special..GLSceneVCL\Packages\CPPoutput.optset file with paths to necessary directories:

 

 

 9. To properly use  DesignEditors, DesignIntf, ToolsAPI and VCLEditors units as specified in RAD Studio Help you need to add the compiler option -LUDesignIDE  on the Compiling page in Project Options:

 

 

 

10. Compile GLScene’s packages.

Right click mouse button on the first line of GLSceneVCL group in Project Manager window and execute “Compile All From Here” command. Then choose to “Install” every DesignTime package of visual components to RAD Studio palette pages.

(Note: the package GLSsceneVCL_SDL.dpk could be used in run time but weirdly clashes with new development environment in design time, so don’t install it, GLSceneVCL_DWS requires additional libraries to be installed).

 

After installation you may start to work with GLScene’s library demos for Delphi and C++Builder in ..GLSceneVCL\Samples directory.
