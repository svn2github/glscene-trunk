Installation of GLScene
into Embarcadero RAD Studio

The Instruction for installing GLScene library into Embarcadero RAD Studio to work with Delphi & C++Builder includes next steps:

1. Download source codes of GLScene from the URL of repository https://svn.code.sf.net/p/glscene/code/trunk to a SVN directory on your disk, e.g. D:\Library\SVN_GLScene. Use TortoiseSVN client or embedded subversion control system for RAD Studio environment in menu File | Open From Version Control… to check out the code.

You may get also the whole current Snapshot of the trunk from code page http://sourceforge.net/p/glscene/code/HEAD/tree/  or download archive zip files with previous releases of GLScene project at the page http://sourceforge.net/projects/glscene/files/

 

2. Make a copy of the trunk in a separate directory, e.g. in the working directory D:\GLScene, to prevent original sources from occasional changes. You may skip this step if you don’t need to update the code from SVN repository further.

 

3. Run setup.bat in ..\GLScene\Source\external directory as administrator to copy auxiliary dynamic libraries into C:\Windows\System32 and C:\Windows\SysWOW64 directories to support 3D sounds, shaders and physics (BASS, FMOD, OpenAL, SDL, CG, ODE, Newton). Otherwise you will need to have ones in every application directory when your exe file calling an appropriate dll. Note: on 64-bit OS Windows 10 you may need to copy *.dlls manually.

 

4.  Setup Delphi Library Paths in Options dialog. If you are installing GLScene in RAD Studio for the first time, then it’s necessary to create a new environmental variable GLSCENEDIR as shown in the next screenshot:

 

Then open Delphi Options Library page

 

and add the $(GLSCENEDIR)\Source directories to search files by compiler:

 

 

 

5. Setup C++ Options for include files for  Classic C++ Compiler:

Open Options dialog for Paths and Directories and add path into “Ordered list of Include paths” to get header HPP files be found by GLScene’s CPP applications:

 

6. Setup C++ Options for libraries files. Add to “Ordered list of Library paths” the path to lib/bpi files for GLScene’s CPP applications

 

 

 

7. Open the GLScene.groupproj file in your directory $(GLSCENEDIR)\Packages\ using menu item File|Open Project…(Ctrl+F11). In Project Manager window you will find the next list of projects with *.bpl extensions:

 

 

8. Check the options of Delphi Compiler to choose “Generate all C++Builder files (including package libs)”. By default all *dpk Projects have this Options saved in *dproj files:

 

 

 9. To properly use  DesignEditors, DesignIntf, ToolsAPI and VCLEditors units as specified in RAD Studio Help you need to add the compiler option -LUDesignIDE  on the Compiling page in Project Options:

 

 

10. Compile GLScene’s packages.

Right click mouse button on the first line of GLScene group in Project Manager window and execute “Compile All From Here” command. Then choose to “Install” every DesignTime package to RAD Studio component palette.

(Note: GLScene_SDL.dpk  and GLScene_DWS packages require additional libraries to be installed and could not be used in current version of GLScene).

 

After installation you are ready to use GLScene’s library and run demos for Delphi & C++Builder in $(GLSCENEDIR)\Samples directory.
